# Memo V0.9 Source-aware / Turn-aware 最终推进方案

> 重要级别：最高  
> 用途：明天换电脑后，优先阅读此文档，并让 Agent 严格按本文档逐步执行，直到 Memo V0.9 source-aware 版本完整落地。  
> 当前状态：设计已定型，尚未开始真实数据迁移。  
> 核心红线：当前已有 `memo.db` 只做完整备份归档，未经确认不得在旧库上继续写入、聚合、重算或删除。

---

## 0. 执行总原则

### 0.1 这次改造的目标

将 Memo 从旧的“碎片化 message memory”升级为：

```text
真实 Agent 会话 source_session
  → 会话内具体 turn / tool / final answer 证据
  → 用户问题/任务 episode
  → 最小长期记忆 memory_unit
  → 跨会话图谱 / Space / Persona / Todo
```

这不是简单优化旧数据，也不是把会话压成摘要。

核心目标：

```text
从源头重新建立干净、可追溯、可跨会话融合的长期记忆系统。
```

### 0.2 绝对禁止

- 禁止直接改写当前真实数据库 `E:\memo\memo\data\memo.db`。
- 禁止在旧碎片 `memory_units` 上继续做模型总结式聚合并当作新方案。
- 禁止把 Memo 生成的标题冒充 Agent 原始会话标题。
- 禁止把工具日志、模型中间话术、阶段性自述作为主长期记忆。
- 禁止在没有 dry-run 报告和备份的情况下执行 apply。
- 禁止把 `.env`、`memo.db`、`data/`、`backups/`、`logs/` 打包进 release。

### 0.3 可接受

- 可以保留 Git 历史中的旧探索提交，但 main 主线不能继续依赖旧方向代码。
- 可以生成 fallback display title，但必须标记 `title_source = generated_fallback`。
- 可以从旧库查询历史，但旧库默认只作为归档证据源。
- 可以在数据库副本上做实验。

第一批先做只读探测，不写 Memo 数据库。

### 4.2 HanaAgent 探测规则

```text
session-titles.json：jsonl path → UI 会话标题
```

```text
detect_manifest_db()
```

## 7. Turn → Episode → Memory 提取规则

### 7.1 turn 解析

保留全部原始 turn，但对长期记忆筛选。

角色：

```text
user
assistant
tool
system
```

标记：

```text
is_tool_call
is_tool_result
is_final_answer
```

### 7.2 episode 切分

新 episode 触发：

- 新用户问题。
- 用户切换任务。
- 用户纠正方向。
- 出现独立决策/事故/发布结果。

不触发新 episode：

- 模型中间总结。
- 工具调用结果。
- “我接下来要检查...”这类过程话术。

### 7.3 memory_unit 提取

一个 episode 可生成多条最小长期记忆。

长期记忆类型：

```text
fact
decision
preference
event
reasoning
todo_hint
technical_fix
risk_rule
```

speaker_scope：

```text
user_claim
assistant_result
tool_observation
decision
mixed
```

## 8. Hebbian / 图谱重构

旧 Hebbian 的问题不是机制无效，而是输入过脏。

新规则：

```text
同 episode 高质量共现：强关系
同 source_session 普通共现：中关系
跨多个真实 source_session 反复出现：长期稳定关系
用户显式偏好/决策：高权重
工具日志/模型中间话术：低权重或不参与
generated fallback title：不作为真实会话关系证据
```

新增关系类型建议：

```text
co_episode
co_session
cross_session_repeat
supports_decision
contradicts
supersedes
belongs_to_space
mentions_entity
evidence_turn
```

图谱默认主节点：

```text
memory_unit
feature_tag
entity
source_session
episode
space
persona_assertion
todo
```

默认隐藏：

```text
tool log
assistant process narration
low-value fragment
```

## 9. Space / Persona / Todo 接入新数据

### 9.1 Space

Space 不能被会话限制。

绑定层级：

```text
memory_unit → Space：主绑定
episode → Space：辅助绑定
source_session → Space：来源提示，不作为唯一边界
```

一个会话可包含多个 Space，一个 Space 可跨多个会话。

### 9.2 Persona

人格只从高质量长期记忆抽取。

优先：

```text
user_claim
decision
preference
跨会话重复行为
```

禁止：

```text
tool log
assistant process narration
generated fallback title
```

### 9.3 Todo

Todo 候选必须保留证据链：

```text
todo → memory_unit
todo → episode
todo → source_turn evidence
```

## 14. 最终完成标准

Memo V0.9 source-aware 版本完成时，必须满足：

1. 旧库已备份归档，不被破坏。
2. 新库可从 Agent 原始会话导入。
3. 每条长期记忆能追溯到 source_session 和 source_turn。
4. 会话标题来源可解释，generated fallback 不冒充真实标题。
5. 一个会话可拆多个 episode，一个 Space 可跨多个会话。
6. Hebbian / 图谱基于新数据规则。
7. Persona / Todo / Space 都使用高质量 memory_unit。
8. Dashboard 能展示来源链路。
9. 测试通过。
10. release 包不含数据库、日志、env、备份。
